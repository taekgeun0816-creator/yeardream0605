## 프로젝트 개요
- **동일한 학생/수강 데이터를 4가지 DB 정규화 단계**  
  (비정규화 → 1NF → 2NF → 3NF)  
  에 각각 저장,  
  사용자가 직접 **갱신/삽입/삭제 시 어떤 이상(Anomaly)이 발생하는지 비교**
- **이상 현상 핵심 비교**
  - 비정규화/1NF : 갱신·삽입·삭제 이상 구조적으로 존재
  - 2NF : 삽입/삭제는 정상, **학과 갱신 이상 잔존(이행 종속)**
  - 3NF : 모든 이상 **해결(정상)**
- 실습 DB 예제:
  - `비정규화.sql`, `1NF.sql`, `2NF.sql`, `3NF.sql` (학생·수강·학과 사례)

---

## 폴더 및 주요 파일 역할

- **`app.py`**
  - Streamlit UI 구현
  - 4개 탭 (비정규화, 1NF, 2NF, 3NF) 제공
  - 입력 폼/테이블 조회/이상 상태 뱃지/SQL 실행문 표시
- **`norm_db.py`**
  - SQLite 로직 담당
  - 4단계별 테이블 생성/초기데이터 적재
  - 각 시나리오별 SQL 실행, 이상/불일치 판정
- **`normalization.db`**
  - 실행 시 자동생성되는 SQLite 파일
  - 4단계 테이블 및 샘플 데이터 포함
- **`pyproject.toml`**
  - uv 프로젝트 세팅
  - python, streamlit, pandas 등 의존성 명시
- **`uv.lock`**
  - 패키지 버전 고정(lock) 파일
- **`README.md`**
  - 실습 안내서(본 문서)

## 실습 시나리오 (Streamlit 탭별)

- 각 탭에서 입력 값 → 버튼 클릭
- 4단계 DB에 **동일 의미 작업 동시 실행**
- 실행 후 이상 여부 및 데이터 불일치 즉시 확인

- **주요 시나리오**
  - ➊ **학과 건물 변경**  
    - 학과 코드 & 새 건물명 입력  
    - 갱신 이상(Anomaly) 여부 관찰
  - ➋ **교수 연구실 변경**  
    - 과목 코드 & 새 연구실 입력
  - ➌ **수강 등록**  
    - 학번/과목/성적 입력(삽입 이상 체험)
  - ➍ **학생 수강 삭제**
    - 학번 입력(삭제 이상 체험)

- 탭 상단의 "이상 상태 표" 해설

  | 단계       | 갱신(학과) | 갱신(교수) | 삽입  | 삭제  | 종합     |
  |-----------|------------|------------|-------|-------|---------|
  | 비정규화  | 존재      | 존재      | 존재 | 존재 | 이상 구조 |
  | 1NF       | 존재      | 존재      | 존재 | 존재 | 이상 구조 |
  | 2NF       | **잔존**  | 정상      | 정상 | 정상 | 이행 종속 |
  | 3NF       | 정상      | 정상      | 정상 | 정상 | **정상** |

  - 각 실습 실행 시 **현 데이터 불일치도 자동 체크**

## 4단계별 테이블 구조

- **비정규화**:  
  - `student_course_all`  
  - 한 셀에 다중값 저장 (예: `CS301,CS302`)
- **1NF**:  
  - `student_course_1nf`  
  - 원자값, (student_id, course_id) 복합 PK
- **2NF**:  
  - `student_2nf`, `course_2nf`, `enrollment`  
  - 부분 함수 종속 제거
- **3NF**:  
  - `department`, `professor`, `student`, `course`, `enrollment_3nf`  
  - 이행(Transitive) 함수 종속 제거
